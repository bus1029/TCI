# TCI 개인 OKR 초안

- **Objective 1.** AI 분석 서비스를 안정적으로 제공할 수 있도록 백엔드 방어 로직과 진단 체계를 구축한다.
    1. Key Result 1. LLM 호출이 포함된 핵심 분석 API 전체에 대해 입력 검증, 예외 처리, 타임아웃, 표준 오류 응답 규격을 적용한다.
    2. Key Result 2. 요청 수신, 내부 처리, 응답 반환 단계별 추적 로그와 오류 분류 기준을 정의하고 핵심 API 2개 이상에 반영한다.
    3. Key Result 3. 핵심 분석 API 2개 이상에 대해 실패 상황별 응답 정책과 복구 가능한 예외 처리 방식을 구현한다.

- **Objective 2.** AI 분석 서비스를 안정적으로 운영할 수 있는 백엔드 배포 및 운영 체계를 구축한다.
    1. Key Result 1. 핵심 백엔드 서비스 2개 이상에 대해 빌드, 테스트, 배포 절차를 문서화하고 자동 실행 가능한 형태로 구성한다.
    2. Key Result 2. 주요 API 및 배치 작업의 상태를 확인할 수 있는 모니터링 항목과 알림 기준을 정의하고 적용한다.
    3. Key Result 3. 장애 발생 시 원인 확인과 대응이 가능하도록 로그, 에러, 배포 이력을 연계해 추적할 수 있는 운영 절차를 수립한다.

- **Objective 3.** 외부 AI 엔진을 서비스에 안정적으로 연결하고 교체 가능한 백엔드 연동 구조를 정립한다.
    1. Key Result 1. LLM 또는 분석 엔진 연동에 대한 요청/응답 규격, 타임아웃, 오류 변환 기준을 정의한다.
    2. Key Result 2. 엔진별 차이를 흡수할 수 있는 공통 연동 계층을 설계하고 핵심 기능 1건 이상에 적용한다.
    3. Key Result 3. 엔진 설정 변경 또는 교체 시 서비스 레이어 수정 범위를 최소화할 수 있도록 연동 구조와 검증 절차를 문서화한다.
