# TCI 전체 기능 리스트 - 최종 산출물용

- 원본: [Notion - TCI 전체 기능 리스트 - 최종 산출물용](https://www.notion.so/334c21d54f4280478809ff500f937299)
- 기준 시각: 2026-04-09
- 정리 기준: 원본 표의 `구분 > 영역 > 기능명 > 설명 > 상세 기능 리스트` 구조를 Markdown 문서로 재구성
- 주의: 이 문서는 외부 제품 사실 요약이 아니라 내부 요구사항 카탈로그다. 따라서 GitHub, Jira, Confluence 관련 항목은 실제 연동 설계 단계에서 서비스별 인증 방식, 이벤트 모델, 배포 형태, 제약 조건을 별도로 확정해야 한다.

## 데이터 수집

### 데이터 소스 연동

#### 코드 저장소 연동
- 설명: Git 저장소와 GitHub/GitLab/Bitbucket 같은 Git 호스팅 플랫폼과 연동하여 코드베이스와 변경 이력을 수집하고 분석 가능한 코드 스냅샷을 생성한다.
- 상세 기능:
  - Git 기반 레포지토리 연결(SSH/HTTPS)
  - 분석 대상 브랜치/태그 선택
  - 제외/포함 경로 및 파일 타입 설정
  - 코드 변경 이벤트 감지(Commit, Push, PR)
  - Webhook 기반 실시간 이벤트 수신
  - 커밋 메타데이터 수집(작성자, 메시지, 시간)
  - 저장소 기준 코드 스냅샷 생성
  - 웹, IDE 동기화
  - 멀티 레포지토리 관리

#### 티켓 시스템 연동
- 설명: Jira 등 외부 티켓 관리 시스템과 연동하여 코드 외 업무 맥락 정보를 수집하고 분석에 활용한다.
- 상세 기능:
  - Jira OAuth/API 기반 연동
  - 프로젝트/스페이스 선택
  - 이슈 유형 및 상태 필터링
  - 이슈 메타데이터 수집(제목, 설명, 작성자, 상태, 링크)

#### 문서 시스템 연동
- 설명: Confluence 등 외부 문서 시스템과 연동하여 문서 메타데이터, 본문, 변경 이력을 수집한다.
- 상세 기능:
  - Confluence 연동
  - 프로젝트 선택
  - 문서 메타데이터 수집(제목, 작성자, 수정 이력)
  - 문서 본문 콘텐츠 수집
  - 문서 변경 이력 동기화

#### 실제 연동 설계 시 확인해야 할 제약
- 설명: GitHub/Jira/Confluence와 같은 외부 시스템을 실제로 붙일 때는 공통 기능 목록 외에 인증, 이벤트, 배포 방식, API 제약을 서비스별로 확정해야 한다.
- 상세 체크리스트:
  - GitHub: GitHub App, OAuth App, PAT 중 어떤 인증 모델을 지원할지 결정
  - GitHub: GitHub.com과 GHES를 모두 지원할지 결정
  - GitHub: Push, PR, Review, Check Run, Issue, Discussion 등 어떤 이벤트를 수집할지 결정
  - GitHub: webhook signature 검증과 재시도/중복 수신 처리 정책 정의
  - GitHub: API rate limit, secondary rate limit, 대용량 저장소 스캔 정책 정의
  - GitHub: Git LFS, submodule, monorepo, fork repository 처리 정책 정의
  - Jira: Cloud와 Server/Data Center 중 어떤 배포 형태를 지원할지 결정
  - Jira: OAuth와 API Token 중 어떤 인증 모델을 지원할지 결정
  - Jira: Project, Issue Type, Workflow Status, Custom Field 매핑 정책 정의
  - Confluence: Cloud와 Server/Data Center 중 어떤 배포 형태를 지원할지 결정
  - Confluence: Page, Space, Attachment, Version History 중 어떤 범위를 수집할지 결정
  - 공통: 데이터 보존 기간, 증분 동기화 기준, 스냅샷 재현 정책, 권한 실패 시 동작 정의

### 파일 관리

#### 파일 업로드 관리
- 설명: 사용자가 직접 관리하는 코드 또는 문서를 업로드하여 데이터로 수집하고 버전 단위로 관리한다.
- 상세 기능:
  - 코드베이스 추가(ZIP 업로드)
  - 매뉴얼, 사내 규정 업로드
  - 스펙 문서 업로드(기획서, 요구사항 정의서)
  - 문서 버전 관리(버전별 비교 지원)
  - 웹, IDE 동기화

### 개발자 도구(플러그인)

#### 로컬 코드 변경 수집
- 설명: IDE 플러그인을 통해 로컬에서 발생한 코드 변경을 캡처하여 서버로 전송하고 분석에 활용한다.
- 상세 기능:
  - IDE 플러그인에서 코드 변경 Diff 캡처
  - 로컬 변경 코드 스냅샷 생성
  - 분석 요청 실행
  - TCI 서버로 코드 스냅샷 전송

### 데이터소스 관리

#### 연동 상태 관리
- 설명: 데이터 수집 대상 시스템의 연동 상태를 모니터링하고 정상 동작 여부를 관리한다.
- 상세 기능:
  - 연동 대상 목록 조회
  - 채널 연결 상태 확인
  - 마지막 동기화 시각 확인
  - 오류 로그 조회

#### 데이터소스 스냅샷 관리
- 설명: 수집된 데이터소스를 특정 시점 기준으로 고정하여 분석 결과의 일관성과 재현성을 보장한다.
- 상세 기능:
  - 메타데이터 스냅샷 저장
  - 스냅샷 버전 이력 관리
  - 특정 시점 기준 분석 데이터 재현
  - 스냅샷 간 변경 비교

#### 인증 정보 관리
- 설명: 데이터소스별 인증 정보를 등록하고 안전하게 저장 및 관리한다.
- 상세 기능:
  - OAuth 인증 정보 등록
  - API 토큰 등록
  - SSH Key 등록
  - 인증 정보 암호화 저장
  - 인증 정보 삭제

### 데이터 소스 관리

#### 동기화 정책 관리
- 설명: 데이터 소스별 동기화 방식과 주기를 설정하여 안정적인 데이터 수집과 운영을 지원한다.
- 상세 기능:
  - 동기화 정책 설정
  - 증분 동기화 설정
  - 전체 스냅샷 동기화 설정
  - 동기화 주기 설정

## 시스템 구조 분석

### 구조 데이터 생성

#### 기술 스택 분석
- 설명: 코드 및 빌드 설정 파일을 분석하여 프로젝트의 언어, 프레임워크, 런타임 환경을 식별하고 구조 분석, 의존성 분석, 인터페이스 분석, 영향 분석에 필요한 기반 정보를 생성한다.
- 상세 기능:
  - 프로그래밍 언어 식별(Java, Python 등)
  - 프레임워크 식별(Spring, Express, Django 등)
  - 빌드 시스템 식별(Maven, Gradle, npm 등)
  - 실행 환경 구성 분석(application.yml, config 등)
  - API 스타일 식별(REST, gRPC 등)
  - 메시지 브로커 사용 식별(Kafka, RabbitMQ 등)
  - 외부 SDK 및 라이브러리 식별
  - 프로젝트 스택 메타데이터 생성

#### 컴포넌트 구조 분석
- 설명: 소스 코드를 분석하여 시스템의 모듈 구조, 컴포넌트 경계, 아키텍처 레이어 및 주요 진입점을 식별하고 구조 분석, 의존성 분석, 구조 시각화에 필요한 핵심 구조 정보를 생성한다.
- 상세 기능:
  - 모듈 구조 분석
  - 패키지/디렉터리 트리 기반 구조 분석
  - 트리 스캔 및 모듈 경계 자동 식별
  - 논리적 모듈 단위 식별
  - 소스 루트 기준 자동 모듈 분할
  - 컴포넌트 경계 식별
  - 빌드 단위(Maven/Gradle 등) 기준 컴포넌트 범위 식별
  - 서브프로젝트 기준 컴포넌트 구분
  - 아키텍처 레이어 분류
  - Controller, Service, Repository, Entity 등 패턴 기반 자동 분류
  - 엔트리포인트 식별
  - main, API 엔드포인트, 배치 작업, 스케줄러 등 진입점 식별
  - 레이어/컴포넌트 구조 식별 규칙 관리
  - 기본 규칙셋(Spring, ProFrame, ProObject 등) 제공
  - 사용자 정의 규칙 추가 및 수정

#### 코드 메타데이터 생성
- 설명: 코드 구조 및 설정 정보를 분석하여 클래스, 메서드, 설정, 의존성 등의 메타데이터를 생성하고 구조 분석, 의존성 분석, 영향 분석에 필요한 정보를 제공한다.
- 상세 기능:
  - 클래스/인터페이스/메서드 시그니처 추출
  - 클래스/인터페이스/메서드의 이름, 파라미터, 반환 타입 등 기본 구조 정보 수집
  - 어노테이션 기반 의미 분석
  - 어노테이션/데코레이터 파싱
  - 역할 및 동작 의도를 메타데이터로 태깅
  - 설정 기반 구성 정보 분석
  - XML, YAML, properties 등 설정 파일 분석
  - 코드 외 설정 정보 수집
  - 의존성 메타데이터 생성
  - 빌드 설정 기반 라이브러리 및 버전 정보 수집
  - 데이터 매핑 정보 분석
  - DB 연결 정보 및 스키마 매핑 메타데이터 수집
  - 코드와 DB 테이블 간 매핑 관계 생성
  - 엔티티 역할 분류
  - DTO, VO, Entity, Utility 등 엔티티의 아키텍처적 역할을 메타데이터 기반으로 자동 분류

#### 아키텍처 관계 분석
- 설명: 구조 메타데이터를 기반으로 모듈, 컴포넌트, 클래스 간의 호출, 의존, 상속, 구현, 참조 관계를 분석하여 의존성 분석 및 영향 분석에 활용 가능한 구조화된 관계 데이터를 생성한다.
- 상세 기능:
  - 호출/의존/상속/구현/참조 관계 분석
  - 관계 유형 분류 및 라벨링
  - 관계 방향성 태깅(단방향/양방향)
  - 직접/간접 관계 구분
  - 모듈/컴포넌트/클래스 단위 관계 정규화
  - 관계 그래프 데이터 생성
  - 관계 데이터 API 및 구조화 포맷 제공

### 서비스 구조 분석

#### 서비스 인터페이스 분석
- 설명: 시스템 간 API 호출, 메시지 브로커, 외부 SDK 등을 분석하여 서비스 간 인터페이스를 식별하고 인터페이스 분석, 시스템 경계 영향 분석, 영향 분석에 필요한 구조 정보를 생성한다.
- 상세 기능:
  - API 호출 구조 분석
  - HTTP 기반 API 호출 식별
  - RestTemplate, WebClient, axios 등 분석
  - RPC 호출 식별(gRPC, Thrift 등)
  - 메시지 브로커 연동 식별(Kafka, RabbitMQ 등)
  - SDK 호출 식별(결제 SDK, 인증 SDK 등)
  - 서비스 엔드포인트 식별
  - API 엔드포인트 자동 추출
  - 인터페이스 매핑
  - 서비스 간 인터페이스 연결 관계 생성
  - 호출 대상, 프로토콜, 엔드포인트 메타데이터 생성

#### 멀티 프로젝트 인터페이스 매핑
- 설명: 복수 프로젝트 간 인터페이스를 분석하여 전체 시스템의 서비스 간 연결 구조를 생성하고 인터페이스 분석, 시스템 경계 영향 분석, 구조 시각화에 필요한 관계 정보를 제공한다.
- 상세 기능:
  - 프로젝트 간 호출 지점 식별
  - 외부 호출 및 서비스 경계 식별
  - 인터페이스 크로스 매칭
  - 복수 프로젝트의 엔드포인트를 대조하여 자동 매칭
  - 연결 대상이 불명확한 엔드포인트를 별도 표시하여 누락/오류 점검 지원
  - 서비스 연결 구조 생성
  - 서비스 간 의존 구조를 토폴로지 형태로 생성
  - 의존 방향 분석
  - 서비스 간 호출 방향 및 흐름 구성

## 구조 시각화 및 진단

### 구조 시각화

#### 아키텍처 맵 생성
- 설명: 분석된 구조 데이터와 아키텍처 관계 정보를 기반으로 시스템의 전체 구조를 시각화하여 구조 이해와 영향 분석에 활용할 수 있는 시각적 정보를 제공한다.
- 상세 기능:
  - 시스템 레벨 아키텍처 다이어그램 자동 생성
  - 서비스/모듈/계층 간 관계 자동 시각화
  - 마이크로서비스 및 레이어 경계 식별
  - 외부 시스템 연동 지점 표시
  - 주요 진입점 및 핵심 실행 경로 강조
  - 다이어그램 다운로드 및 공유 지원

#### 시스템 구조 탐색
- 설명: 구조 분석 결과를 기반으로 시스템 구조, 서비스 관계, 흐름을 탐색하고 관련 코드 및 문서와 연결하여 구조 이해, 구조 진단, 영향 분석을 지원한다.
- 상세 기능:
  - 모듈/서비스 구조 시각화
  - 서비스 호출 흐름 탐색
  - 비즈니스 흐름 시각화
  - API/DB/서비스 관계 탐색
  - 관련 문서 및 코드 위치 연결

### 구조 진단

#### 호출 관계 분석

  - 데이터 계약 영향 분석(DTO, 이벤트 스키마 변경 등)
  - 시스템 간 영향 대상 목록 생성
  - 시스템 간 영향 요약 생성

#### 테스트 영향 분석
- 설명: 코드 변경이 영향을 줄 수 있는 테스트 케이스와 커버리지를 분석하여 테스트 대상을 도출하고 변경 검증 범위의 효율적 설정을 지원한다.
- 상세 기능:
  - 변경 코드 기준 영향 테스트 식별
  - 테스트 코드/자산 수집
  - 커버리지 데이터 수집
  - 테스트-코드 매핑
  - 테스트 커버리지 분석
  - 누락 테스트 탐지

#### 리스크 분석
- 설명: 코드 구조와 변경 이력을 기반으로 잠재 리스크를 정량적으로 분석하고 위험 수준을 산정하여 우선 점검 대상 선정과 변경 의사결정을 지원한다.
- 상세 기능:
  - 구조적 복잡도 기반 리스크 점수 산정
  - 변경 영향 범위 크기 기반 위험 등급 분류
  - 서비스 경계 외부(타 시스템) 영향 범위 탐지
  - 이력 기반 고위험 모듈(핫스팟) 목록 제공
  - 우선 점검 대상 자동 추천
  - 리스크 요소 상세 설명

### 결과 시각화

#### 영향 결과 시각화 및 비즈니스 요약
- 설명: 변경 전후 주요 변경점, 영향 범위, 지표 변화를 시각화하고, 기술 지표를 비즈니스 언어로 번역하여 대시보드 및 리포트 원본을 생성한다.
- 상세 기능:
  - 변경 영향 범위를 목록, 그래프, 요약 형태로 시각화
  - 영향 범위 리포트 Export 지원(일정 협의, 보고용)
  - 변경 전후 코드/품질 지표 변화 시각화
  - 코드 지표 비교(복잡도, 결합도 등)
  - 시스템 품질 지표 비교(응집도, 순환 복잡도 등)
  - 변경안(A vs B) 적용 시 영향 범위 수치 비교(모듈 수, API 수, 트랜잭션 수) 대시보드 제공
  - 기술 지표를 비즈니스 언어로 번역한 영향 요약 제공

- 상세 기능:

  - 객체 역할 설명

- 상세 기능:

  - 후속 질문 컨텍스트 유지
  - 접근 권한 기반 응답 제어

## 워크플로우 자동화

### 워크플로우

#### PR 본문 자동 생성
- 설명: 코드 변경 내용을 기반으로 PR 본문을 자동 생성하고 변경 근거 및 영향 분석 결과를 반영하여 리뷰와 협업을 지원한다.
- 상세 기능:
  - PR Diff 기반 문서 초안 생성(코드 스니펫 포함)
  - 변경 근거 및 영향 범위 자동 서술
  - 코드 위치 링크 포함 문서 생성
  - 비즈니스 규칙 변경 분석 결과 반영(기획서/스펙 항목과 연결)

- 상세 기능:
  - PR 변경 요약 생성
  - 변경 의도 및 설계 맥락 요약
  - 영향 서비스, 모듈 식별
  - 인터페이스 정의 변경 및 비즈니스 규칙 변경 식별
  - PR 관련 맥락 정보 반영
  - 영향을 받는 모듈에 대해 분석된 리스크 포인트 삽입

#### CI 연동
- 설명: CI 파이프라인과 연동하여 코드 변경 이벤트 기반 분석과 자동화 작업을 실행한다.
- 상세 기능:
  - PR/MR/파이프라인 이벤트 연계

  - 분석 결과 CI 파이프라인 리포트 통합

#### 분석 자동화
- 설명: 스케줄 및 이벤트 기반으로 분석을 자동 실행하고 결과 리포트를 생성 및 전달한다.
- 상세 기능:
  - 스케줄 기반 분석 실행
  - 스케줄 기반 주기적 갭 분석 실행(매주 스펙-코드 갭 분석)
  - 분석 결과 리포트 생성 및 배포
  - 이벤트 기반 분석 자동 트리거(PR, MR, 문서 변경 등)
  - 분석 결과 알림(Slack, 이메일 등)

#### PR 컨텍스트 연동
- 설명: PR 변경과 관련된 티켓 및 문서를 자동으로 연결하여 코드 변경과 함께 맥락 정보를 제공하고 리뷰 및 변경 이해를 지원한다.
- 상세 기능:
  - PR 관련 티켓 자동 탐색 및 연결
  - PR 관련 문서 자동 탐색 및 연결
  - 코드 변경과 문서 맥락 통합 제공

#### 비즈니스 리포트 생성
- 설명: 영향 분석 결과를 기반으로 비즈니스 관점의 리포트를 생성하고 지정된 포맷과 주기에 맞춰 자동으로 발송한다.
- 상세 기능:
  - 이해관계자 보고용 리포트 자동 생성(경영진, 고객사 등)
  - 리포트 포맷 커스터마이징 지원(항목 선택, 상세도 조절)
  - 정기 스케줄 기반 리포트 자동 생성 및 갱신(주간/월간 등)
  - 특정 중요 이벤트 발생 시 리포트 생성
  - 외부 채널 자동 발송(Slack, 이메일, 메신저 연동)
