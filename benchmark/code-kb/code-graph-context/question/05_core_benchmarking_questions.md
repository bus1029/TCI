# CodeGraphContext 개발 노하우 참고 질문

이 문서는 지식 베이스 구축 시 CodeGraphContext 제품의 핵심 동작 방식과 강점을 파악하고, 우리 제품에 적용할 수 있는 요소들을 정리한 질문 모음입니다.

## 1. 지식 모델링과 고유성 보장 방안

- **Q. 우리 제품의 지식 베이스는 문서 청크 중심인가, 심벌 중심인가, 그래프 중심인가, 벡터 중심인가, 하이브리드형인가?**
  - 참고할 노하우
    - 단순 텍스트 임베딩보다 명확한 심벌(AST)과 그래프 기반의 구조화가 코드 분석 에이전트에게 유리함
  - 아키텍처 의사결정 시, CodeGraphContext처럼 로컬 환경에서 트리시터(Tree-sitter)를 활용해 코드를 구조적으로 파싱할지 검토 필요함
- **Q. 우리 제품은 어떤 엔티티(함수, 클래스, 모듈 등)를 명시적으로 저장할 것인가?**
  - 참고할 노하우
    - 파일 단위를 넘어 함수, 클래스, 메서드 단위까지 엔티티를 세분화하여 모델링하면 구조 질의가 강력해짐
  - 어떤 메타데이터(엔티티)를 명시적으로 테이블(또는 노드)에 저장할지 명확한 스키마 설계 요망
- **Q. 우리 제품은 엔티티 간 어떤 관계(CALLS, INHERITS 등)를 저장할 것인가?**
  - 참고할 노하우
    - 코드 검색과 RAG를 넘어 "이 코드가 어디서 호출되는가(CALLS)?"를 추적하려면 엣지(관계) 추출이 필수적임
  - 에이전트가 코드의 영향 범위와 의존성을 역추적할 수 있게 해 주는 기반 시스템으로 채택 고려함

## 2. 고강도·다단계 인덱싱 파이프라인

- **Q. 파일 변경 시 즉각적으로 지식베이스를 갱신하는 기능(Watcher)과 증분 갱신 로직을 어떻게 탑재할 것인가?**
  - 참고할 노하우
    - IDE의 잦은 저장 이벤트를 묶어 처리(Debounce)하는 방식 점검
    - 콜 그래프 무결성을 위해 증분 및 릴링킹을 수행하는 것이 운영 성능의 핵심임
  - 지식 베이스가 항상 최신 코드를 반영하도록 유지보수성과 DB 성능 밸런스를 맞추는 아키텍처 과제임
- **Q. 인덱싱 소요 시간을 줄이기 위해 지식 베이스를 묶음(.cgc 번들)으로 내보내고 가져오는 기능을 지원할 것인가?**
  - 참고할 노하우
    - 대형 레포지토리 파싱 시간을 우회하기 위해 한 번 인덱싱된 상태를 팀원 간 단일 파일로 주고받는 훌륭한 전략임
  - 인덱싱 초기 지연 시간을 없애고 팀원 간 지식을 매우 쉽게 공유할 수 있는 핵심 기능으로 채택 검토함

## 3. 검색 엔진 및 저장소 아키텍처

- **Q. 우리는 어떤 데이터베이스를 사용할 것이며, 로컬 DB와 원격 대형 DB 간 전환 가능한 추상화 레이어를 제공할 것인가?**
  - 참고할 노하우
    - 기본 환경으로는 제로 설정(Zero-config)의 KùzuDB를 제공함
    - 엔터프라이즈 환경 시 Neo4j 등으로 갈아끼울 수 있게 설계하면 도입 장벽을 크게 낮춤
  - 사용자 초기 설정 부담을 없애면서 엔터프라이즈 환경에도 대응할 수 있는 DB 드라이버 추상화 구조 도입함
- **Q. 코드 원문을 LLM API 등 외부로 전송하지 않고 완전한 로컬 보안 망 내에서 오프라인 파싱과 검색을 수행할 수 있는가?**
  - 참고할 노하우
    - B2B 및 대기업 도입 시 무단 원문 전송 이슈를 완전한 로컬 파이프라인으로 원천 차단함
  - 클라우드 API 의존도를 낮추고 로컬 내에서만 도는 "코드 유출 리스크 제로" 지식 베이스 구축 시 필수 요건이 됨

## 4. 지능형 에이전트 도구 인터페이스

- **Q. 우리 지식 베이스는 챗 인터페이스에 머무를 것인가, 아니면 MCP 형태로 노출하여 자율 AI 에이전트 도구(Tool Calling)가 될 것인가?**
  - 참고할 노하우
    - 지식 베이스를 MCP 서버로 구축하면 여러 클라이언트와 에이전트 생태계에 플러그인처럼 삽입될 수 있음
  - 타겟이 보조적인 챗봇인지, 리팩토링 미션을 수행하는 주도적 에이전트인지 성격을 명확히 규정함
- **Q. 에이전트가 데드코드 분석, 복잡도 파악, 호출 계층 역추적 등 아키텍처 질의를 어느 수준까지 자유롭게 행할 수 있는가?**
  - 참고할 노하우
    - 단순 검색에서 벗어나 "Caller와 Callee 트리를 뽑아달라"는 등 고난도 질의 API 확장이 가능함
  - 실질적인 코드 분석 에이전트 작동을 위해 확장할 도구(API) 목록의 설계 지점을 제공함
- **Q. 에이전트 편집 환각을 막아줄 실제 기기의 절대 경로(`path`)와 정확한 위치(`line_number`)가 응답에 포함되는가?**
  - 참고할 노하우
    - 검색 결과 노드 객체에 절대 경로와 정확한 코드 줄 범위를 강제로 동봉해 반환함
  - 에이전트가 "편집과 리팩토링 능력을 갖춘 RAG"로 거듭나게 해주는 가장 근본적인 설정임
