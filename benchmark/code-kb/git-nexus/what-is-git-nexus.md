# GitNexus 한눈에 이해하기

## 이 문서의 목표

이 문서는 GitNexus를 처음 접하는 사람이 짧은 시간 안에 아래 내용을 이해하도록 돕기 위해 작성했다.

- GitNexus가 어떤 프로젝트인지
- 왜 일반적인 AI 코딩 도구나 코드 검색만으로는 부족한 문제를 풀려고 하는지
- 이 프로젝트가 내세우는 강점이 무엇인지
- 핵심 개념이 무엇인지
- 저장소 안에서 어떤 부분이 실제 제품 가치의 중심인지

대상 독자는 AI 기술을 깊게 배우지 않았고, MCP나 knowledge graph 같은 용어도 낯설 수 있는 사람을 기준으로 잡았다. 어려운 용어가 나오면 가능한 한 쉬운 말로 바로 풀어서 설명한다.

## 한 줄 설명

GitNexus는 AI 코딩 도구를 위해 코드베이스를 지식 그래프로 바꾸는 프로젝트다.

더 정확히 말하면, Git 저장소 안의 파일, 함수, 클래스, 호출 관계, import 관계, 실행 흐름을 미리 분석해서 그래프로 저장한 뒤, 그 결과를 CLI, MCP, HTTP API, 웹 UI에서 다시 쓰게 해 주는 시스템이다.

## 왜 이런 프로젝트가 필요한가

많은 AI 코딩 도구는 코드를 다룰 때 주로 아래 두 방식을 쓴다.

- 현재 열려 있는 파일이나 일부 코드 조각만 보고 답하기
- `grep`, IDE 검색, 문자열 검색으로 관련 코드를 찾아보기

이 접근은 작은 수정에는 도움이 되지만, 실제 개발 작업에서는 몇 가지 한계가 있다.

- 어떤 함수를 고치면 어디까지 영향이 퍼지는지 알기 어렵다
- 이름이 비슷한 함수가 여러 개 있으면 어떤 것이 진짜 대상인지 헷갈리기 쉽다
- 텍스트 검색으로는 "실행 흐름"을 한 번에 보기 어렵다
- 여러 파일에 걸친 호출 관계를 사람이 직접 머릿속에서 이어 붙여야 한다
- AI가 일부 코드만 보고 수정하다가 다른 곳을 깨뜨릴 가능성이 높다

GitNexus는 바로 이 지점을 겨냥한다. 이 프로젝트는 단순히 "관련 텍스트를 찾는 것"이 아니라, "이 코드가 어떤 구조 안에 있고", "누가 누구를 호출하며", "여길 바꾸면 어디가 깨질 수 있는지"까지 같이 보여 주려 한다.

## GitNexus가 푸는 문제

예를 들어 어떤 AI 코딩 도구가 로그인 버그를 고친다고 가정해 보자.

- `login` 함수가 실제로는 여러 서비스와 유틸을 호출한다
- API 라우트, 인증 미들웨어, 세션 저장 로직이 서로 연결돼 있다
- 테스트 코드와 다른 기능도 같은 함수를 재사용하고 있다

단순 검색 도구는 관련 파일과 문자열은 찾을 수 있다. 하지만 아래 질문에 안정적으로 답하기는 어렵다.

- 지금 이 함수를 바꾸면 직접 영향을 받는 호출자는 누구인가
- 이 로직은 어떤 실행 흐름의 일부인가
- 비슷한 이름의 함수 중 어느 것이 진짜 수정 대상인가
- 이름을 바꾸면 어느 파일까지 같이 수정해야 하는가
- 지금 내 변경이 어떤 기능 경로를 깨뜨릴 수 있는가

GitNexus는 이런 정보를 `심볼`, `관계`, `커뮤니티`, `프로세스` 단위로 저장하고, 변경 영향 분석과 구조 검색 도구로 꺼내 쓸 수 있게 만든다.

## 이 프로젝트의 강점

### 1. 코드 구조를 미리 계산한 그래프

GitNexus의 가장 큰 강점은 코드를 질문할 때마다 즉석에서 추측하지 않고, 먼저 저장소 전체를 분석해 그래프로 만들어 둔다는 점이다.

- 파일 구조 기록
- 함수, 클래스, 메서드 추출
- 호출 관계와 import 관계 연결
- 상속과 구현 관계 연결

이 덕분에 AI는 단순 텍스트 조각이 아니라 "구조가 있는 코드베이스"를 바탕으로 일하게 된다.

### 2. 실행 흐름 중심 탐색

GitNexus는 검색 결과를 단순 파일 목록으로만 주지 않는다.

- 어떤 기능 흐름에 속하는지
- 어떤 단계에서 호출되는지
- 관련 심볼이 어떤 묶음으로 연결되는지

즉 검색 결과가 "이 파일도 관련 있어 보인다"에서 끝나지 않고, "이 기능이 실제로 어떻게 흘러가는지"까지 따라가기 쉬운 형태로 나온다.

### 3. 변경 영향 분석

GitNexus는 단순 검색 도구가 아니라 변경 리스크를 보는 도구이기도 하다.

- 특정 심볼을 바꿨을 때 직접 영향받는 호출자 확인
- 간접적으로 영향받을 수 있는 심볼 추적
- 관련 실행 흐름과 기능 영역 확인
- Git diff 기준 변경 영향 분석

그래서 "무엇을 고쳐야 하나"뿐 아니라 "고치면 무엇이 깨질 수 있나"까지 같이 다룰 수 있다.

### 4. 하나의 인덱스를 여러 인터페이스에서 재사용

GitNexus는 한 번 분석한 결과를 여러 방식으로 활용한다.

- 터미널에서 쓰는 CLI
- AI 도구가 연결하는 MCP 서버
- 브라우저 UI와 연결되는 HTTP 서버
- 그래프를 시각적으로 보는 웹 UI

즉 별도 제품이 여러 개 있는 것이 아니라, 같은 분석 결과를 여러 사용 방식으로 노출하는 구조에 가깝다.

### 5. 로컬 우선 설계와 에이전트 통합

GitNexus는 가능한 한 로컬에서 동작하도록 설계돼 있다.

- 코드 분석 결과를 저장소 내부 `.gitnexus/`에 저장
- 전역 레지스트리로 여러 저장소를 관리
- Cursor, Claude Code, OpenCode 같은 도구와 연결
- 로컬 서버나 MCP로 외부 에이전트에 노출

이 방향 덕분에 보안상 민감한 코드베이스에도 비교적 적용하기 쉽고, AI 도구의 "코드베이스 이해 부족" 문제를 바로 보완하기 좋다.

## 핵심 개념

GitNexus를 이해할 때 가장 중요한 개념은 다섯 가지다.

| 개념 | 의미 | 왜 중요한가 |
| --- | --- | --- |
| Repository | 분석 대상 Git 저장소 | 모든 작업의 시작점 |
| Symbol | 파일, 함수, 클래스, 메서드 같은 코드 요소 | 탐색과 수정의 기본 단위 |
| CodeRelation | 호출, import, 상속, 구현 같은 관계 | 코드 구조를 연결하는 핵심 |
| Community | 관련 심볼 묶음 | 기능 영역을 큰 단위로 이해하는 데 유용 |
| Process | 실행 흐름 또는 call chain | 실제 동작 경로를 추적하는 데 중요 |

### Repository

Repository는 GitNexus가 분석하는 실제 코드 저장소다.

예시

- 프론트엔드 앱 저장소
- 백엔드 API 저장소
- 모노레포

GitNexus는 저장소를 분석한 뒤 결과를 `.gitnexus/`에 저장한다. 그래서 나중에 다시 질문할 때 처음부터 매번 전체 코드를 새로 읽지 않아도 된다.

### Symbol

Symbol은 그래프 안의 코드 요소다.

예시

- 파일
- 함수
- 클래스
- 인터페이스
- 메서드

AI 코딩 작업 관점에서는 "무엇을 읽고", "무엇을 고치고", "무엇이 영향을 받는지"를 판단하는 기본 단위라고 보면 된다.

### CodeRelation

CodeRelation은 심볼 사이의 구조적 관계다.

예시

- 함수 A가 함수 B를 호출한다
- 파일 A가 파일 B를 import 한다
- 클래스 A가 클래스 B를 상속한다
- 클래스 A가 인터페이스 B를 구현한다

중요한 점은 이 관계가 단순 선 연결이 아니라, AI가 영향 분석과 구조 검색을 할 수 있게 만드는 핵심 데이터라는 점이다.

### Community

Community는 서로 관련이 깊은 심볼 묶음을 뜻한다.

쉽게 말해, 인증 관련 코드끼리, 결제 관련 코드끼리, 검색 관련 코드끼리 자동으로 묶어 보여 주는 상위 단위다. 저장소가 커질수록 파일 하나하나보다 이런 기능 묶음이 더 유용해진다.

### Process

Process는 실제 실행 흐름 또는 call chain에 가까운 개념이다.

예를 들면 아래 같은 상황에 맞다.

- 사용자가 API를 호출한 뒤 서비스 계층을 거쳐 DB에 도달하는 흐름
- 버튼 클릭 이후 여러 함수가 순서대로 실행되는 흐름
- 특정 기능이 시작점부터 끝점까지 이어지는 경로

즉 GitNexus는 코드 조각만 모아 두는 것이 아니라, "이 기능이 실제로 어떻게 움직이는가"를 다룬다.

## GitNexus는 어떻게 동작하나

가장 중요한 흐름은 `Analyze`와 `질의/탐색` 두 가지다.

### 1. Analyze

새 저장소를 분석할 때는 대체로 아래 순서로 처리된다.

1. Git 저장소인지 확인한다
2. 파일 구조를 스캔한다
3. Tree-sitter로 함수, 클래스, 메서드 같은 심볼을 추출한다
4. import, call, 상속 관계를 연결한다
5. 관련 심볼을 community로 묶는다
6. 실행 흐름을 process로 정리한다
7. 결과를 KuzuDB와 검색 인덱스에 저장한다
8. `.gitnexus/meta.json`과 전역 레지스트리를 갱신한다

핵심은 "코드를 질문할 때마다 즉석에서 읽는 것"이 아니라, 먼저 구조화된 인덱스를 만들어 둔다는 점이다.

### 2. 질의와 탐색

분석이 끝나면 GitNexus는 여러 질문에 답할 수 있게 된다.

- `query`
    - 관련 기능 흐름을 검색
- `context`
    - 특정 심볼의 호출자, 피호출자, 소속 흐름 확인
- `impact`
    - 변경 영향 범위 분석
- `detect_changes`
    - 현재 Git diff가 어떤 흐름에 영향을 주는지 확인
- `rename`
    - 여러 파일에 걸친 이름 변경 보조
- `cypher`
    - 고급 그래프 질의

여기서 MCP는 AI 도구가 외부 도구를 호출하는 공통 연결 방식 정도로 이해하면 충분하다. GitNexus는 이 MCP를 통해 AI 코딩 도구가 그래프 질의를 직접 호출하게 만든다.

## 일반적인 코드 검색과 어떻게 다른가

| 항목 | 일반적인 코드 검색 또는 AI 코딩 도구 | GitNexus |
| --- | --- | --- |
| 기본 단위 | 파일, 문자열, 현재 열린 코드 조각 | repository, symbol, relation, process |
| 주요 관심사 | 관련 텍스트 찾기 | 구조, 실행 흐름, 영향 범위 파악 |
| 변경 처리 | 사람이 직접 영향 범위를 추적 | impact, detect_changes로 보조 |
| 검색 결과 | 파일 목록, 라인 매치 | 기능 흐름, 심볼, 관계, 커뮤니티 |
| 관계 표현 | 암묵적 | 그래프 구조로 명시적 표현 |
| 다중 저장소 지원 | 도구마다 제각각 | 전역 레지스트리 기반 지원 |

GitNexus는 코드 검색을 완전히 대체하는 도구라기보다, "AI가 코드를 더 안전하게 이해하고 수정하게 돕는 구조 인텔리전스 레이어"라고 보는 편이 정확하다.

## 저장소는 어떻게 구성돼 있나

처음 보는 사람은 저장소를 세 부분으로 나눠 이해하면 된다.

### `gitnexus`

프로젝트의 핵심 패키지다.

여기에 실제 제품 가치가 가장 많이 들어 있다.

- CLI 명령
- 저장소 인덱싱 파이프라인
- KuzuDB 적재
- MCP 서버
- 로컬 HTTP 서버
- 검색과 영향 분석 로직

### `gitnexus-web`

브라우저 기반 UI다.

여기는 GitNexus 그래프를 눈으로 탐색하고, 코드와 관계를 시각적으로 확인하고, Graph RAG 스타일 채팅을 해 볼 수 있는 프론트엔드에 가깝다. ZIP 업로드, Git clone, 로컬 서버 연결 세 경로를 지원한다.

### `eval`

SWE-bench 기반 평가 하네스다.

이 부분은 "GitNexus를 붙인 AI 에이전트가 실제 소프트웨어 수정 작업을 더 잘하는가"를 실험하기 위한 영역이다. 제품 그 자체라기보다 검증 장치에 가깝다.

## 누가 이 프로젝트를 좋아할 가능성이 큰가

아래 같은 문제를 가진 팀이라면 GitNexus가 특히 잘 맞을 수 있다.

- 큰 코드베이스를 AI 코딩 도구와 함께 다루는 팀
- 함수 하나를 고칠 때 영향 범위를 빠르게 알고 싶은 팀
- 리팩터링과 rename을 더 안전하게 하고 싶은 팀
- 코드 구조를 웹 UI로 시각적으로 탐색하고 싶은 팀
- AI 에이전트에 "코드베이스 이해" 능력을 붙이고 싶은 팀
- SWE-bench 같은 벤치마크로 도구 효과를 검증하고 싶은 팀

## 반대로 주의할 점

GitNexus는 강력하지만, 모든 상황에 가장 단순한 해법은 아니다.

- 런타임 동작을 100퍼센트 보장하는 도구는 아니다
- 인덱스가 오래되면 최신 코드와 어긋날 수 있다
- 브라우저 UI는 큰 저장소에서 메모리 한계를 만날 수 있다
- 임베딩과 고급 검색은 더 많은 시간과 자원을 쓸 수 있다
- 언어 지원 범위는 parser와 추출 품질의 영향을 받는다

즉 GitNexus는 "구조를 이해하는 문제"에는 강하지만, 실제 실행 결과와 테스트를 완전히 대신해 주는 도구는 아니다.

## 이 프로젝트를 한 문장으로 다시 말하면

GitNexus는 AI 코딩 도구가 코드베이스의 구조, 실행 흐름, 변경 영향 범위를 더 정확히 이해하도록 돕는 코드 지식 그래프 플랫폼이다.
