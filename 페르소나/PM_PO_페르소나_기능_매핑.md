# PM/PO 페르소나 기능 매핑

> 페르소나: **박피엠** (PM/PO / 레거시 시스템 환경(SI/공공/금융) 비즈니스 요구사항 정의 및 개발팀 협업)
> 출처: [PM/PO 페르소나 정의](https://www.notion.so/31ac21d54f4280129069dc4ee7bc652f)

---

## 매핑 기준

페르소나의 **TCI 도입 전 불만사항 → 목표 → 니즈** 흐름을 기준으로,
각 기능이 어떤 상세 기능을 가져야 하는지 도출했다.

| 불만사항 | 목표 | 니즈 |
|---|---|---|
| 비즈니스 규칙이 코드에 어떻게 구현됐는지 확인할 수단이 없음 | 기획서·스펙과 구현 갭을 개발자 없이 직접 확인 | 코드 → 자연어 스펙 자동 추출, 업무 언어 질의 → 근거 링크 포함 답변 |
| 요구사항 변경 영향 분석이 개발자 인터뷰에 의존해 병목 | 변경 영향 범위를 사전에 파악해 근거 기반 일정 협의 | 영향 모듈·API·트랜잭션 자동 추적, 변경안 비교, 역방향 추적 |
| 코드와 기획서 사이의 갭을 QA·릴리즈 이후에야 발견 | 기획서·스펙과 구현 갭을 개발자 없이 직접 확인 | 스펙 문서와 코드 불일치 자동 탐지, Jira 티켓 연결 인수 검증 |
| 레거시 시스템의 현행 업무 규칙 파악에 며칠씩 소요 | 현재 시스템의 업무 규칙을 자연어로 탐색 가능 | 추출된 규칙을 도메인별 카탈로그화, 팀 자산으로 관리 |

## 기능 매핑 결과

| **카테고리** | **기능명** | **설명** | **상세 기능** |
|---|---|---|---|
| 데이터 수집 | Data Sources | 코드/문서/티켓 등 데이터 소스를 등록하고 연동 상태를 관리한다 | • 코드베이스 추가 (ZIP 업로드/Git SSH/HTTPS)<br>• 스펙 문서 업로드 (기획서, 요구사항 정의서, PDF/DOCX/Confluence 등)<br>• 티켓 플랫폼 연동 (Jira 등)<br>• Knowledge Hub(Confluence) 연동<br>• 연동 상태 모니터링 및 동기화 주기 관리<br>• 수집 범위 설정 (브랜치, 경로, 파일 타입 등)<br>• Webhook 기반 실시간 변경 감지 (push, PR, 스펙 업데이트 이벤트)<br>• 문서 버전 관리 (스펙 버전별 비교 지원) |
| 시스템 구조 분석 | Codebase Explanations | 정적 분석으로 모듈/컴포넌트를 자동 발견하고, 모듈별 설명 문서를 생성한다 | • 모듈/컴포넌트 자동 발견 및 분류<br>• 모듈별 파일 목록/콜 트리 제공<br>• 기술 설명/비즈니스 룰/다이어그램/흐름 문서 제공<br>• 업무 도메인 단위로 모듈 그룹화 (기술 레이어 단위가 아닌 업무 기능 단위로)<br>• 비기술자가 이해 가능한 모듈 역할 설명 자동 생성<br>• 업무 도메인 맵 생성 (전자결재, 회원, 결제 등 업무 영역별 구조 요약)<br>• 모듈별 관련 Jira 티켓, 요구사항 연결 |
| 자연어 기반 질의 응답 | Ask Swimm (AI) / Ask Kodesage | 자연어로 코드/구조/도메인을 탐색하고, 컨텍스트 답변(다이어그램 포함)을 제공한다 | • 자연어로 코드/구조/도메인 탐색<br>• 다이어그램 포함 답변<br>• 답변을 문서화해 지식 베이스화<br>• 프롬프트 질의 응답<br>• 답변 평가 (Good/Bad/코멘트)<br>• 컨텍스트/우선순위/태깅/강제 설정<br>• 각 규칙마다 코드 위치 링크가 포함된 답변 제공<br>• 비즈니스 규칙 카탈로그 기반 정확한 답변 생성<br>• 질의 이력 저장 및 팀 공유 |
| 제어/데이터 흐름 분석 | 비즈니스 규칙 추출 (Create specs) | 코드에서 비즈니스 규칙을 추출해 비기술자도 이해 가능한 자연어 스펙으로 제공한다 | • 비즈니스 규칙 자동 추출<br>• 스펙 문서 형태 제공<br>• 도메인 로직 자연어 설명/가시화<br>• 규칙과 코드 위치 연결 (클릭 시 해당 코드로 이동)<br>• 규칙 간 충돌/중복 탐지<br>• 비기술자가 이해 가능한 형태로 변환 (기획서 스펙과 비교 가능한 포맷)<br>• 도메인별 규칙 자동 분류 (서명 처리/결재 흐름/감사 로그 등)<br>• 규칙 변경 이력 추적 |
| 지식 자산화 | Knowledge Base Construction | 분석 결과를 구조화해 지식 모델/그래프를 구성하고 재탐색 가능하게 만든다 | • 구조/의존성 메타데이터 저장<br>• 탐색 가능한 지식 모델 구성<br>• 수집 데이터를 정규화된 지식 노드로 변환<br>• 코드, 문서, 티켓 간 교차 참조 연결<br>• 지식 검색 및 재탐색을 위한 인덱싱<br>• 팀 단위 지식 자산 공유 및 접근 권한 관리<br>• 코드 변경 감지 시 관련 지식 노드 자동 갱신 |
| 지식 자산화 | 비즈니스 규칙 관리 | 코드에서 추출된 비즈니스 규칙을 도메인별 카탈로그로 관리한다 | • 추출된 비즈니스 규칙을 도메인별 카탈로그로 분류, 관리<br>• 규칙 상태 관리 (활성/비활성/deprecated)<br>• 규칙 버전 이력 추적 (언제, 어떻게 변경됐는지)<br>• 팀 내 규칙 검색 및 공유<br>• 기획서 스펙 항목과 규칙 매핑 |
| 영향도 분석 | 문서와 코드 간 불일치 분석 | 기술 문서와 실제 코드 간 불일치를 탐지한다 | • 업로드한 스펙 문서(API 스펙/매뉴얼/README)와 구현 코드의 차이 식별<br>• Jira 티켓 Done 처리 상태와 실제 구현 간 갭 탐지<br>• 갭 항목별 심각도 분류 (구현 누락/의도 불일치/부분 구현)<br>• 갭 탐지 리포트 자동 생성 (항목마다 코드 위치 링크 포함)<br>• 리포트 내보내기 (팀 공유/인수 검증용) |
| 영향도 분석 | Change Impact Analysis | 변경 Diff를 기준으로 영향 범위를 구조/트랜잭션 관점에서 추적한다 | • 스펙/업무 규칙 변경 시 영향받는 모듈, API, 트랜잭션 자동 추적<br>• 변경 범위에 따른 영향 모듈 수, API 수, 트랜잭션 건수 정량 제공<br>• 서비스 경계를 넘는 영향 위험 신호 표시<br>• 영향 범위 호출 그래프 시각화<br>• 비즈니스 로직 변경의 역방향 추적 (어느 코드 구간이 수정돼야 하는지)<br>• 영향 범위 리포트 내보내기 (일정 협의, 보고용) |
| 영향도 분석 | Trade-off 시각화 | 사용자가 고민 중인 변경안에 대한 트레이드오프 보고서를 생성한다 | • 변경 전후 주요 변경점 요약<br>• 변경 영향 범위 정리<br>• 지표 변화(예: 성능) 비교<br>• 변경안 A vs B 영향 범위 수치 비교 (모듈 수, API 수, 트랜잭션 수)<br>• 장단점 및 대안 제시<br>• 의사결정 근거 리포트 자동 생성<br>• 기술 지표를 업무 영향 언어로 번역 (예: "14개 모듈 영향 → 일정 N주 추가 예상") |
| 워크플로우 자동화 | Documentation as Code (PR to Doc) | PR 변경사항을 기반으로 문서 초안을 만들고 AI 본문 생성 여부를 선택할 수 있다 | • PR diff 기반 문서 초안 생성 (스니펫 포함)<br>    ◦ 변경 근거 및 영향 범위 자동 서술<br>    ◦ 코드 위치 링크 포함 문서 생성<br>    ◦ 업무 규칙 변경 여부 명시 (기획서/스펙 항목과 연결)<br>    ◦ 인수 검증 체크리스트 자동 생성 (PM이 확인할 항목 목록)<br>    ◦ 관련 Jira 티켓 링크 자동 삽입<br>• AI 본문 컨텍스트 자동 생성 옵션<br>• 스니펫만 넣고 수동 작성 옵션 |
| 워크플로우 자동화 | Docs Studio | 코드베이스 기반 정형 문서를 템플릿으로 생성하고 변경을 반영하며 승인/수정 흐름을 제공한다 | • 템플릿 기반 문서 생성 (데이터 소스 지정)<br>• 문서 목록/상세 조회<br>• 업무 도메인별 문서 구조 정의 및 관리 (전자결재, 결제, 회원 등)<br>• 코드 변경 감지 시 관련 문서 자동 업데이트 알림<br>• PM이 문서 검토 후 승인/수정하는 흐름 지원<br>• 문서 버전 관리 및 이력 추적<br>• 팀 공유 및 접근 권한 관리 |
| 워크플로우 자동화 | Issue Analysis | 티켓 플랫폼과 연동해 이슈에 대한 수정 제안 코멘트를 자동 생성, 게시한다 | • Jira 등 티켓 플랫폼과 연동해 이슈 관련 코드 구간 자동 연결<br>• 티켓 플랫폼 목록 조회<br>• 티켓 상세 조회<br>• 자동 코멘트 (파일 영향 범위/수정 제안)<br>• 이슈 관련 코드 변경 이력 연결 (어떤 커밋으로 처리됐는지)<br>• 이슈 완료 후 PM 인수 검증 알림 |
| 워크플로우 자동화 | Analysis Automation | 스케줄/이벤트 기반으로 분석 실행을 자동화하고 결과 리포트를 생성한다 | • 스케줄 기반 분석 실행<br>    ◦ 스케줄 기반 주기적 갭 탐지 실행 (예: 매주 스펙 vs 코드 갭 탐지)<br>• 분석 결과 리포트 생성 및 배포<br>• 이벤트 기반 영향 분석 자동 트리거 (스펙 문서 업로드, PR 등록, 코드 머지 등)<br>• 분석 결과 알림 (Slack, 이메일 등) |
| 워크플로우 자동화 | 비즈니스 리포팅 | 기술 변경 내용을 비즈니스 언어로 번역하여 다이어그램과 자연어 요약으로 구성된 보고 자료를 생성한다 | • 기술 변경 내용을 업무 영향 언어로 번역 (예: "결제 모듈 변경 → 월 거래 N만 건 영향 가능성")<br>• 핵심 트랜잭션 흐름을 비기술자도 이해 가능한 다이어그램 + 자연어 요약으로 구성<br>• 이해관계자 보고용 리포트 자동 생성 (경영진, 고객사 등)<br>• 리포트 포맷 커스터마이징 지원 (항목 선택, 상세도 조절)<br>• 정기 보고 자료 자동 갱신 (코드 변경 내용 반영)<br>• 변경 전후 업무 영향 지표 비교 포함 |
