# 기본 기능 도출

> Must Have(11개) + Should Have(21개) 총 32개 기능이 동작하기 위해 반드시 필요하지만, 기능 목록에는 명시되지 않은 **플랫폼/인프라 레벨 기능**을 도출한 결과입니다.

## 기본 기능 상세

### 1. 사용자 인증 / 인가 (Authentication & Authorization)

제품에 접근하는 모든 사용자와 외부 시스템의 신원을 확인하고, 권한 범위를 제어하는 기능

**하위 기능**
- 로그인 / 로그아웃
- SSO 연동 (SAML, OIDC)
- 역할 기반 접근 제어 (RBAC) — 관리자, 분석가, 뷰어 등
- API 키 / 토큰 발급 및 관리
- 세션 관리

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Ask Swimm / Ask Kodesage (사용자별 질의 이력) | Must |
| Enterprise Operation (대규모 조직 운영) | Must |
| 보안 인증 및 배포 (SOC2, SSO, 감사 로그) | Should |
| External Integration (API 인증) | Should |
| 알림/요약 (사용자별 구독) | Should |

---

### 2. 프로젝트 / 워크스페이스 관리

분석 대상을 프로젝트 단위로 구성하고, 레포지토리 연결·멤버 관리·분석 설정 등을 제공하는 기능

**하위 기능**
- 프로젝트(워크스페이스) 생성 / 수정 / 삭제
- 레포지토리 연결 관리 (Git URL, 브랜치 설정)
- 멤버 초대 / 역할 할당
- 프로젝트별 설정 (분석 대상 언어, 제외 경로, 브랜치 정책 등)
- 분석 대상 애플리케이션 식별 및 관리

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Source Code Ingestion (분석 대상 레포 등록) | Must |
| Data Sources (코드/문서/티켓 소스 등록) | Must |
| Static Structure Analysis (어떤 프로젝트를 분석할지) | Must |
| 프로젝트별 맞춤 학습 (프로젝트 단위 최적화) | Should |
| Analysis Automation (프로젝트 단위 스케줄) | Should |

---

### 3. 데이터 영속화 / 저장소

분석 결과, 스냅샷, 메타데이터, 문서 등 제품이 생성하는 모든 데이터를 저장하고 버전 관리하는 인프라

**하위 기능**
- 분석 스냅샷 저장 / 버전 관리 (시점별 비교 가능)
- 분석 결과 메타데이터 영속화
- 생성 문서 저장 / 이력 관리
- 시계열 데이터 관리 (지표 변화 추이 등)
- 대용량 코드베이스 스토리지

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Knowledge Base Construction (지식 모델 저장) | Must |
| Change Impact Analysis (분석 결과 저장/비교) | Must |
| 비즈니스 규칙 추출 (추출 결과 저장) | Must |
| Docs Studio (문서 저장/버전 관리) | Should |
| Trade-off 시각화 (변경 전후 데이터 비교) | Should |
| Change Coupling (이력 데이터 저장) | Should |

---

### 4. 웹 UI / 대시보드 프레임워크

분석 결과, 그래프, 문서, 질의응답 등을 사용자에게 보여주는 화면 체계. 제품의 모든 기능이 **사용자에게 전달되는 최종 접점**

**하위 기능**
- 분석 결과 뷰어 (목록, 상세, 필터링)
- 인터랙티브 그래프/다이어그램 렌더링 엔진
- 채팅/질의 인터페이스 (Ask 기능용)
- 문서 에디터 / 뷰어
- 리포트 / 요약 화면
- 반응형 레이아웃 / 내비게이션

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Ask Swimm / Ask Kodesage (채팅 인터페이스) | Must |
| Change Impact Analysis (영향 범위 표시) | Must |
| 아키텍처 맵 생성 (다이어그램 렌더링) | Should |
| Dependency Analysis (의존성 그래프 표시) | Should |
| Trade-off 시각화 (비교 화면) | Should |
| Risk Insight (리스크 목록/설명 표시) | Should |
| Docs Studio (문서 편집/조회) | Should |

---

### 5. API 게이트웨이

내부 서비스 간 통신과 외부 시스템 연동을 위한 API 인터페이스 계층

**하위 기능**
- REST API 엔드포인트 제공 (분석 결과 조회, 트리거 등)
- 웹훅 수신 (Git 이벤트, CI/CD 이벤트)
- 웹훅 발신 (알림, 외부 시스템 연계)
- API 인증 / Rate Limiting
- API 문서 (OpenAPI 등)

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| CI/CD 연동 (파이프라인/PR 이벤트 수신) | Must |
| Enterprise Operation (외부 시스템 연계) | Must |
| External Integration (REST API 제공) | Should |
| 변경 감지 및 요약 (웹훅으로 변경 수신) | Should |
| Analysis Automation (이벤트 기반 분석 트리거) | Should |
| 알림/요약 (Slack 등 외부 알림 발신) | Should |

---

### 6. 비동기 작업 처리 (Task Queue)

코드 분석, 문서 생성, AI 추론 등 시간이 오래 걸리는 작업을 비동기로 실행하고 상태를 관리하는 인프라

**하위 기능**
- 작업 큐잉 (우선순위 기반)
- 비동기 작업 실행 / 워커 관리
- 작업 상태 추적 (대기 → 실행중 → 완료/실패)
- 실패 시 재시도 / 알림
- 동시 실행 제어 (대규모 분석 시)

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Source Code Ingestion (대규모 코드 수집) | Must |
| Static Structure Analysis (전체 코드베이스 분석) | Must |
| Enterprise Operation (대규모 분석 실행) | Must |
| Analysis Automation (스케줄 기반 분석) | Should |
| 자동 문서/가이드 생성 (AI 문서 생성) | Should |
| Codebase Explanations (모듈별 AI 설명 생성) | Should |
| Dependency Analysis (전체 의존성 그래프 구성) | Should |

---

### 7. 감사 로그 / 시스템 모니터링

사용자 활동과 시스템 상태를 기록·추적하여 보안 감사와 운영 안정성을 지원하는 기능

**하위 기능**
- 사용자 활동 로그 (로그인, 조회, 변경 등)
- 시스템 이벤트 로그 (분석 실행, 오류 등)
- 감사 추적 (audit trail) — 누가 언제 무엇을 했는가
- 시스템 헬스 모니터링 (서비스 상태, 리소스 사용량)
- 로그 보관 / 내보내기

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Enterprise Operation (안정적 운영) | Must |
| 배포/엔터프라이즈 플랜 (엔터프라이즈 지원) | Must |
| 보안 인증 및 배포 (SOC2/ISO 인증, 감사 로그) | Should |
| Analysis Automation (분석 실행 이력 관리) | Should |

---

### 8. 설정 / 구성 관리

전역·프로젝트·사용자 레벨의 설정값을 관리하고, 각 기능의 동작을 사용자가 커스터마이즈할 수 있게 하는 기능

**하위 기능**
- 시스템 전역 설정 (LLM 모델 선택, 분석 기본값 등)
- 프로젝트별 설정 (분석 범위, 브랜치, 제외 패턴 등)
- 사용자별 설정 (알림 구독, 관심 영역, 언어 선호 등)
- 외부 연동 설정 (Git 인증, Slack 채널, CI/CD 웹훅 URL 등)
- 분석 정책 설정 (자동/수동, 스케줄, 트리거 조건 등)

| 이 기본 기능을 전제하는 상위 기능 (대표) | 우선순위 |
|---|---|
| Data Sources (소스 연동 설정) | Must |
| CI/CD 연동 (웹훅/파이프라인 설정) | Must |
| Ask Swimm / Ask Kodesage (LLM 모델/컨텍스트 설정) | Must |
| 알림/요약 (관심 영역/알림 채널 설정) | Should |
| 프로젝트별 맞춤 학습 (학습 대상/정책 설정) | Should |
| Analysis Automation (스케줄/트리거 설정) | Should |
| 변경 감지 및 요약 (감지 대상/알림 설정) | Should |

## 기본 기능 vs 공통 기능 비교

| 구분 | 공통 기능 | 기본 기능 |
|---|---|---|
| 정의 | 여러 상위 기능이 **반복적으로 사용하는 분석/처리 능력** | 제품 자체가 **동작하기 위한 플랫폼/인프라** |
| 예시 | 코드 파싱 엔진, LLM 파이프라인, 의존성 그래프 엔진 | 인증/인가, 프로젝트 관리, 데이터 저장소, UI |
| 사용자 인식 | 직접 의식하지 않지만 기능의 핵심 동력 | 제품을 쓰는 순간부터 자연스럽게 접하는 기반 |
| 없으면? | 특정 기능군이 구현 불가 | 제품 자체가 동작 불가 |
