# [TCI 정의]

## Tmaxsoft Code Intelligence(TCI)

한문장 정의 (2가지 안)

- “코드·문서 등 분산된 개발 관련 자산을 통합 수집·정규화하고, 코드베이스와 코드 변경을 기준으로 시스템 구조, 영향 범위, 관련 맥락을 분석하여 개발 조직의 이해와 판단을 지원하는 엔지니어링 인텔리전스 시스템”
- “코드·문서·이슈·변경 이력 등 분산된 개발 자산을 통합 수집·정규화하고, 코드베이스와 코드 변경을 기준으로 시스템 구조, 영향 범위, 비즈니스 맥락을 분석하여 개발 조직의 이해·검토·판단을 지원하는 엔지니어링 인텔리전스 시스템”

# [TCI 컨셉]

## 제품 컨셉

- Git Repository, Jira, Confluence, 업로드 문서 등 개발 관련 자산을 하나의 콘솔에서 통합 관리
- 코드베이스 자체를 분석하여 시스템 구조, 모듈 간 관계, 의존성, 비즈니스 흐름, 업무 규칙을 식별·정리
- 코드 또는 코드 변경(PR / 브랜치 / 커밋 / Diff)를 기준으로 영향 범위, 관련 구조, 검토 포인트를 분석·제공
- 분석 결과를 자연어 질의응답, 근거 링크, 리뷰 리포트, 문서 초안 형태로 연결하여 실제 검토·리뷰·설명 행동 지원
- AI Agent가 코드베이스를 직접 이해하기 어려운 환경에서, 작업에 필요한 구조·흐름·규칙·주의사항을 컨텍스트 패키지로 생성하여 제공
- 코드에 내재된 비즈니스 로직과 규칙을 추출·정리하고, 이를 문서화 가능한 형태로 연결하여 개발자와 비개발자 모두가 활용 가능한 지식 자산으로 축적

## 제품 포지셔닝

**Engineering Context Platform + Change Intelligence Platform + Documentation Platform + Business Logic Intelligence**

### 정의

### 1. Engineering Context Platform

코드·문서 등 다양한 개발 자산을 연결하여 시스템 이해와 의사결정에 필요한 맥락 정보 제공

### 2. Change Intelligence Platform

코드 자체의 의존 관계와 영향 범위를 분석하고, 코드 변경이 시스템에 미치는 영향과 리스크까지 함께 분석하여 판단 근거 제공

### 3. Documentation Platform

코드, 구조, 변경 이력, 규칙, 관련 문서를 기반으로 리뷰, 공유, 검증에 활용 가능한 문서 초안 및 설명 자료 생성

### 4. Business Logic Intelligence

코드에 구현된 비즈니스 규칙, 흐름, 조건 분기, 정책 로직을 분석하여 사람이 이해 가능한 형태로 정리하고 구조 이해 및 변경 판단에 활용할 수 있도록 지원

### 역할

- 개발자뿐 아니라 리뷰어, 운영/기획 등 비개발자에게도 이해 가능한 형태로 시스템 구조, 코드 관계, 변경 근거, 비즈니스 로직 설명을 정리해주는 역할 수행
- 코드 변경 영향뿐 아니라 코드베이스 자체의 구조, 의존성, 흐름, 업무 규칙까지 함께 분석하여 시스템 이해와 변경 판단에 필요한 정보를 제공
- 분석 결과를 문서, 리포트, 설명 초안 형태로 연결하여 리뷰, 승인, 인수인계, 요구사항 검증, 보고까지 이어지는 실제 업무 흐름 지원
- 또한 AI Agent가 활용할 수 있는 형태의 컨텍스트를 제공하여 코드 생성, 수정, 리뷰, 리팩토링 시 코드베이스 이해도 향상 지원

# [TCI 개요]

## 1. 비즈니스 목적

- 개발 조직의 분산된 맥락 정보를 통합하는 공통 분석 기반 확보
- 코드 변경 영향 분석 및 리뷰·검토 지원을 통한 개발 생산성 향상
- 복잡한 시스템 구조 및 비즈니스 규칙 이해 비용 절감
- AI 기반 개발 환경에서 에이전트와 경쟁하는 도구가 아니라, 에이전트가 코드베이스를 이해하도록 돕는 인텔리전스 레이어 확보
- PR 승인, 요구사항 변경 검토, 인수 검증 등 개발 의사결정 과정 전반에서 활용 가능한 근거 중심 제품 기반 마련

## 2. 개발 목적 및 배경

### 2.1 개발 목적

- 코드, 티켓, 문서, 사용자 입력 맥락을 연결하는 단일 지식 기반 구축
- 변경 단위 중심의 영향 분석 및 검토 지원 체계 마련
- 비즈니스 규칙과 구조 분석 결과를 실제 업무 흐름에 연결하는 사용자 경험 제공
- 복잡한 시스템의 이해·검토·설명 업무를 표준화할 수 있는 공통 제품 기반 확보
- AI Agent가 작업 시작 시 바로 활용할 수 있는 컨텍스트 번들 생성 체계 마련
- PR 리뷰어가 승인 전에 리스크와 영향 범위를 빠르게 판단할 수 있는 분석·리포트 체계 마련
- PO / PM이 개발자 인터뷰에 의존하지 않고도 비즈니스 규칙, 구현 갭, 요구사항 변경 영향을 확인할 수 있는 기반 마련

### 2.2 개발 배경 및 시장 환경

- 코드, 이슈, 문서, 대화 기록이 서로 다른 도구에 분산되어 개발 맥락 파편화 심화
- 변경 영향 판단이 개인 경험과 특정 인력 의존적으로 운영되는 구조
- 대규모 시스템일수록 구조 이해와 비즈니스 흐름 파악에 높은 진입 비용 발생
- 분석 결과가 존재하더라도 실제 검토·리뷰·설명 행동으로 이어지지 못하는 단절 존재
- AI 코드 생성 도구 확산으로 코드 작성 속도는 빨라졌으나, 코드베이스 전체 구조와 맥락 이해 한계로 인해 잘못된 제안, 중복 구현, 영향 범위 누락 문제 지속 발생
- PR 수 증가와 함께 리뷰어의 승인 부담 확대, 변경 리스크 판단 시간 증가
- 엔터프라이즈 고객 환경에서 구조 이해, 변경 영향 분석, 리뷰 근거 확보, AI Agent 컨텍스트 제공 수요 지속 확대

## 3. 핵심 가치

### 3.1 기업 가치

- 공통 분석 자산 확보
    - 코드 구조, 관계 정보, 비즈니스 규칙, 도메인 용어, 문서·이슈 맥락을 통합한 공통 지식 자산 축적
- 엔터프라이즈 제품 경쟁력 강화
    - 복잡한 대형 시스템을 운영하는 고객에게 구조 이해, 영향 분석, 비즈니스 로직 분석, 문서화 지원 기능 제공
- R&D 효율 향상
    - 공통 연동 구조, 분석 파이프라인, 지식 베이스, 시각화·응답 인터페이스 재사용
- AI 개발 환경 대응력 확보
    - AI Coding Agent 확산 환경에서 코드 생성 자체가 아니라 컨텍스트 공급, 구조 이해, 영향 검증, 근거 제공 영역의 차별화 포지셔닝 확보

### 3.2 사용자 가치

- 개발 맥락 확보 시간 단축
    - 코드 외 문서 등 다양한 개발 자산까지 연결된 단일 탐색 경로 제공
- 시스템 구조 및 코드 이해도 향상
    - 코드베이스의 구조, 의존 관계, 서비스 흐름, 비즈니스 규칙을 분석하여 복잡한 시스템에 대한 이해 지원
- 영향 범위 판단 정확도 향상
    - 코드 자체의 영향 범위와 코드 변경에 따른 시스템 내·외부 영향을 함께 분석하여 판단 근거 제공
- 리뷰 및 검토 생산성 향상
    - PR Diff만으로 파악하기 어려운 영향 범위, 리스크, 검토 포인트, 관련 문서를 함께 제공하여 리뷰 부담 감소
- AI Agent 활용 효율 향상
    - 작업 시작 시 필요한 구조, 흐름, 규칙, 주의사항을 컨텍스트 형태로 제공하여 반복 설명 없이 바로 작업 가능한 상태 지원
- 비즈니스 로직 이해 및 검증 지원
    - 비즈니스 규칙, 조건 분기, 정책 로직을 자연어 설명과 편리한 UI 형태로 제공하여 개발자와 비개발자 모두의 이해 지원
- 요구사항 변경 검토 효율화
    - 스펙, 문서, 티켓과 실제 코드 구현을 연결하여 영향 범위와 구현 갭을 빠르게 확인 가능
- 설명 및 문서화 작업 효율화
    - 구조 설명, 변경 근거, 리뷰 리포트, 문서 초안, 보고 자료 등 설명 가능한 산출물 생성 지원
- 지식 축적 및 재사용
    - 분석 결과, 질의응답, 문서, 규칙 정보를 조직 차원의 지식 자산으로 축적하고 반복 활용 가능
- 근거 기반 의사결정 지원
    - 코드, 문서, 티켓, 정책 링크를 기반으로 구조 이해, 변경 판단, 리뷰 승인, 일정 협의, 인수 검증 시 신뢰 가능한 근거 제공

## 6. User Persona

### Persona 1. 개발자

- 복수의 모듈, 문서, 이슈를 동시에 참고하며 변경 작업 수행
- 관련 코드뿐 아니라 영향받는 서비스, API, 데이터 흐름까지 빠르게 확인 필요
- 기존 담당자가 아니었던 영역에 대해서도 구조, 흐름, 주의사항, 관련 문서를 빠르게 이해할 수 있는 지원 필요
- AI Agent를 활용하는 경우 작업 시작 전 필요한 컨텍스트를 반복 설명 없이 바로 확보할 수 있는 지원 필요

### Persona 2. 리뷰어 / 시니어 개발자

- 짧은 시간 내 다수의 PR을 검토해야 하는 역할
- PR Diff만으로는 파악하기 어려운 비즈니스 규칙 변경, 인터페이스 정의 변경, 시스템 외부 영향 여부를 우선 식별할 필요
- 중요한 변경 포인트와 위험 구간을 먼저 확인하고, 테스트·문서·연관 모듈 누락까지 함께 점검할 수 있는 검토 지원 필요
- 코드 외에도 Jira, Confluence, 설계 문서 등 변경 맥락까지 포함한 승인 근거 확보 필요

### Persona 3. PM / PO

- 요구사항과 실제 구현 사이의 연결 관계를 개발자 도움 없이 확인할 수 있는 수단 필요
- 현재 시스템에 어떤 업무 규칙이 살아있는지 자연어 기반으로 탐색하고, 스펙과 코드 간 불일치를 조기에 확인할 수 있는 지원 필요
- 요구사항 변경 시 영향받는 모듈, API, 트랜잭션을 근거 기반으로 파악하고 일정 협의나 보고에 활용할 수 있는 자료 필요

## 7. User Story

### 개발자

- 변경 전 관련 모듈, 문서, 이슈, 규칙을 함께 확인하며 작업 범위와 주의사항 파악
- 특정 로직, 비즈니스 규칙, 실제 실행 경로의 구현 위치를 빠르게 탐색
- AI Agent 작업 시작 전 구조·흐름·규칙·유사 구현 정보를 컨텍스트로 제공받아 반복 설명 없이 작업 수행
- PR 생성 전 영향 범위, 누락 가능 영역, 검증 포인트를 사전 점검

### 리뷰어

- PR 오픈 시 변경 요약, 영향 범위, 리스크 포인트, 검토 포인트를 우선 확인
- 연관 문서, Jira 티켓, 설계 맥락까지 함께 검토하여 승인 판단 정확도 향상
- 비즈니스 규칙 변경, 인터페이스 정의 변경, 시스템 내·외부 영향 여부를 빠르게 식별
- 테스트·문서·연관 모듈 누락 여부를 사전 확인하고, 필요 시 외부 AI Agent 또는 자동화 도구에 분석 결과 활용

### PM / PO

- 업무 규칙을 자연어 스펙 형태로 확인하고 기획 의도와 실제 구현 간 차이 검토
- 요구사항 변경 시 영향받는 모듈, API, 트랜잭션을 확인하여 일정 협의와 우선순위 판단에 활용
- 스펙 문서, Jira 티켓, 실제 코드 구현 간 연결 관계를 확인하고 인수 검증 및 보고 자료로 활용
