## 공통 기능 상세

### 1. 코드 파싱 / 정적 분석 엔진

코드를 언어별로 파싱하여 AST(추상 구문 트리)를 생성하고, 구조/심볼/관계 메타데이터를 추출하는 핵심 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Static Structure Analysis (컴포넌트/레이어 식별) | Must |
| 비즈니스 규칙 추출 (도메인 로직 추출) | Must |
| Dependency Analysis (호출/의존 관계 분석) | Should |
| Codebase Explanations (모듈 자동 발견) | Should |
| 영향도 분석 (변수/흐름 추적) | Should |
| 리팩토링 (중복 코드 식별) | Should |
| 테스트 (누락 테스트 탐지) | Should |

---

### 2. Git/VCS 연동 및 Diff 처리 엔진

Git 레포지토리를 연결하고, 커밋/PR/브랜치 간 Diff를 계산하며, 변경 이력을 추적하는 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Source Code Ingestion (Git 기반 수집) | Must |
| Change Impact Analysis (변경 Diff 기준 영향) | Must |
| CI/CD 연동 (PR/MR 이벤트 연계) | Must |
| Change Coupling (커밋 간 결합도) | Should |
| 변경 감지 및 요약 (코드/문서 변경 감지) | Should |
| Documentation as Code - PR to Doc (PR diff 기반) | Should |
| Trade-off 시각화 (변경 전후 비교) | Should |
| 코드 수정 사항 제안 (스펙 변경 시 영향) | Should |

---

### 3. 의존성 / 호출 그래프 엔진

파싱 결과를 기반으로 함수→함수, 모듈→모듈, 서비스→DB 등의 호출/의존 관계를 그래프로 구성하고, 경로 탐색·순환 탐지·영향 전파 계산을 수행하는 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Change Impact Analysis (트랜잭션 연계 영향) | Must |
| Dependency Analysis (의존성 그래프, 순환 탐지) | Should |
| 영향 범위 분석 (라이브러리 영향 파일/함수) | Should |
| 영향도 분석 (변수 설정→전파→출력 경로) | Should |
| 코드 수정 사항 제안 (API 변경 시 프론트 영향) | Should |
| 아키텍처 맵 생성 (모듈 간 관계) | Should |
| Risk Insight (복잡도 기반 위험 식별) | Should |

---

### 4. 지식 그래프 / 메타데이터 저장소

분석 결과(구조, 의존성, 호출 관계, 비즈니스 규칙 등)를 구조화된 그래프 형태로 저장하고, 탐색·질의·재사용이 가능하게 하는 저장소. 다른 공통 기능들의 **산출물이 모이는 허브** 역할

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Knowledge Base Construction (지식 모델 구성) | Must |
| Ask Swimm / Ask Kodesage (컨텍스트 기반 답변) | Must |
| Static Structure Analysis (구조 메타데이터 저장) | Must |
| Codebase Explanations (모듈별 정보 조회) | Should |
| 아키텍처 맵 생성 (관계 데이터 조회) | Should |
| 프로젝트별 맞춤 학습 (메타 인텔리전스 저장) | Should |

---

### 5. LLM / AI 추론 파이프라인

LLM을 활용하여 자연어 생성, 질의응답, 요약, 의도 추론, 코드↔자연어 변환 등을 수행하는 파이프라인. **가장 많은 상위 기능이 의존하는 공통 기능**

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Ask Swimm (자연어 질의 → 다이어그램 포함 답변) | Must |
| Ask Kodesage (프롬프트 질의 응답) | Must |
| 비즈니스 규칙 추출 (도메인 로직 자연어 설명) | Must |
| Codebase Explanations (AI 생성 설명) | Should |
| 자동 문서/가이드 생성 (문서 초안 AI 생성) | Should |
| Documentation as Code - Code to Doc | Should |
| Documentation as Code - PR to Doc (AI 본문 생성) | Should |
| 변경 감지 및 요약 (변경 의도 추론) | Should |
| 코드 수정 사항 제안 (영향 분석 → 수정 제안) | Should |
| Trade-off 시각화 (변경점 요약/보고서) | Should |
| 테스트 (시나리오 추천) | Should |
| 리팩토링 (모듈화 후보 제안) | Should |
| Risk Insight (리스크 설명) | Should |

---

### 6. 문서 생성 / 관리 엔진

다양한 소스(코드, PR, 분석 결과)에서 문서를 생성·편집·버전 관리하고, 템플릿 적용 및 승인/수정 흐름을 제공하는 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| 비즈니스 규칙 추출 (스펙 문서 형태 제공) | Must |
| Ask Swimm (답변을 문서화해 지식 베이스화) | Must |
| 자동 문서/가이드 생성 | Should |
| Documentation as Code - Code to Doc | Should |
| Documentation as Code - PR to Doc | Should |
| Docs Studio (템플릿 기반 생성, 승인 흐름) | Should |
| 알림/요약 (요약 리포트 생성) | Should |

---

### 7. 이벤트 감지 / 알림 시스템

코드 변경, PR 생성, 분석 완료 등 이벤트를 감지하고, 조건에 따라 알림 발송 및 후속 워크플로우를 트리거하는 시스템

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| CI/CD 연동 (파이프라인/PR/MR 이벤트 연계) | Must |
| 변경 감지 및 요약 (변경 감지 → 요약/알림) | Should |
| 알림/요약 (관심 영역 기반 알림) | Should |
| Analysis Automation (스케줄/이벤트 기반 분석 트리거) | Should |

---

### 8. 데이터 수집 / 인제스션 파이프라인

Git뿐 아니라 문서, 티켓, 위키 등 다양한 외부 소스에서 데이터를 수집하고, 분석 가능한 형태로 정규화·저장하는 파이프라인

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Source Code Ingestion (코드 스냅샷 수집) | Must |
| Data Sources (코드/문서/티켓/위키 연동) | Must |
| Change Coupling (변경 이력 데이터 수집) | Should |
| 프로젝트별 맞춤 학습 (프로젝트 데이터 수집) | Should |

---

### 9. 분석 실행 / 스케줄링 엔진

분석 작업을 스케줄링·실행·모니터링하고, 대규모 코드베이스에서도 안정적으로 분석을 완료할 수 있도록 관리하는 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Enterprise Operation (대규모 분석 실행 관리) | Must |
| CI/CD 연동 (파이프라인 이벤트 시 분석 실행) | Must |
| Analysis Automation (스케줄 기반 분석) | Should |

---

### 10. 검색 / 컨텍스트 검색 엔진

지식 베이스, 코드, 메타데이터를 대상으로 키워드/시맨틱 검색을 수행하여, AI 질의응답이나 탐색 기능에 관련 컨텍스트를 제공(RAG)하는 엔진

| 이 공통 기능이 필요한 상위 기능 | 우선순위 |
|---|---|
| Ask Swimm (코드/구조/도메인 탐색) | Must |
| Ask Kodesage (컨텍스트/우선순위/태깅) | Must |
| Knowledge Base Construction (탐색 가능한 지식 모델) | Must |
| 리팩토링 (코드 유사도 탐색) | Should |
| 프로젝트별 맞춤 학습 (프로젝트 컨텍스트 활용) | Should |
